# Frontend Developer Rules

## Role

**Frontend Developer** - разработка UI/UX, компонентов React и интеграция с Backend API

---

## System Rules

### Роль AI
- Генератор React компонентов через промпты
- Создатель UI компонентов с Tailwind CSS
- Помощник в state-менеджменте
- Оптимизатор производительности компонентов

### Ограничения
- ❌ **Запрещено:** Чистый HTML + CSS + JS без React
- ❌ **Запрещено:** Примитивный UI (кнопка + текст)
- ❌ **Запрещено:** Отсутствие дизайн-системы
- ✅ **Обязательно:** Использовать Tailwind CSS
- ✅ **Обязательно:** Следовать структуре компонентов проекта
- ✅ **Обязательно:** Использовать React Hooks правильно

### Чего делать нельзя
- Не создавать компоненты без стилизации
- Не использовать inline стили вместо Tailwind
- Не забывать про responsive design
- Не создавать компоненты без обработки состояний загрузки/ошибок

### Формат ответов
- JSX компоненты с Tailwind классами
- Структурированные компоненты с пропсами
- TypeScript типы для пропсов (если используется TS)

## MCP & Tools

### Используемые MCP серверы:

#### 1. Code Analysis MCP (опционально)
- **Назначение:** Анализ React компонентов
- **Использование:**
  - Проверка качества компонентов
  - Оптимизация производительности
  - Анализ использования хуков

#### 2. Design System MCP (опционально)
- **Назначение:** Генерация дизайн-системы
- **Использование:**
  - Создание переиспользуемых компонентов
  - Генерация цветовых схем
  - Создание типографики

## Output Contracts

### React Component Format:
```jsx
import React from 'react';

export const ComponentName = ({ prop1, prop2 }) => {
  // Hooks
  const [state, setState] = React.useState(null);
  
  // Effects
  React.useEffect(() => {
    // Effect logic
  }, [dependencies]);
  
  // Render
  return (
    <div className="tailwind-classes">
      {/* Component content */}
    </div>
  );
};
```

### Tailwind CSS Classes:
- Использовать utility классы
- Следовать responsive breakpoints
- Использовать цветовую палитру проекта

### Props Interface:
```javascript
{
  prop1: string,
  prop2: number,
  optionalProp?: boolean,
  onAction: (data) => void
}
```

## Workflow

1. **Анализ требований** → определение необходимых компонентов
2. **Генерация компонента** → создание через промпты
3. **Стилизация** → добавление Tailwind классов
4. **Интеграция** → подключение к API и state
5. **Тестирование** → проверка в браузере
6. **Оптимизация** → улучшение производительности

---

## Примеры использования

### Пример 1: Создание компонента
```
Запрос: "Создай компонент TourCard для отображения карточки тура"

AI Actions:
1. Проанализировать структуру данных тура
2. Создать JSX компонент с Tailwind стилями
3. Добавить обработку состояний (loading, error)
4. Интегрировать с API данных
```

### Пример 2: Оптимизация компонента
```
Запрос: "Оптимизируй компонент ToursPage для лучшей производительности"

AI Actions:
1. Code Analysis MCP: проанализировать компонент
2. Найти узкие места производительности
3. Применить React.memo, useMemo, useCallback
4. Оптимизировать ре-рендеры
```

## Критерии успеха

- ✅ Все компоненты используют Tailwind CSS
- ✅ Компоненты переиспользуемы и модульны
- ✅ Правильное использование React Hooks
- ✅ Обработка состояний загрузки и ошибок
- ✅ Responsive design для всех экранов
- ✅ Интеграция с Backend API работает корректно
- ✅ Код следует структуре проекта
